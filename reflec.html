Group reflection

In week 11, we have an assignment that requires us to think about the IT project. So the range of this project is nearly limitless. After discussing with the whole team, we decided to create a project called CycleIT, a combination of education and e-commercial website where we want to raise awareness of people about the environment and fund organizations that have the same purpose. But why the environment? This topic is not new but never gets old. We believe that everyone understands that the climate has continued to deteriorate due to human interventions. People might argue that debating and raising awareness about the different forms of pollution is rather dull and not innovative, however, we stand firm that we can contribute a new perspective to the table and create opportunities for the environmental crisis via CycleIT. Our motivation behind this project is when doing some research and reading intensively, we found so many valuable things that can help the environment and we want to bring this to as many people as possible. Like another project, at the initial stage, we will have a bunch of meetings to discuss the project topic and divide missions for each member of the team. We also created a timetable which was included in the Project Proposal so everyone will know what they have to do weekly. Of course, after 2 or 3 days we would have a standup meeting to update the situation with the team and see if anyone had a roadblock during the project or not. We found this was really helpful for everyone to understand how the project is going on. Hiep was assigned to write the project description which was the main part of our report while Quan had the responsibility to describe project outcomes and skills and jobs. The final member Dung was assigned to do the prototype website, and the report website and also write tools and the group reflection part. Doing such a huge amount of work on this project with two other courses was a challenge for all of us, however dividing the mission at the very early stage really helped us to keep track with the timeline of the project. Each member faced a lot of challenges during their own mission, however by sharing with other members, the whole team found a way together to solve each member's problem which not only supports the project but also give us a chance to improve our communication and working as a team skills.
